# 07 — Tax Service (W-9 + 1099)

Produces everything **up to** the tax return: W-9 collection, TIN matching, 1099 eligibility tracking, e-filing of information returns, and CPA-ready Schedule C packets. **Acme Finance does not file income tax returns.**

## Current thresholds (verify each season — store in `tax_thresholds`)

Under the One Big Beautiful Bill Act (OBBBA, signed July 2025):

| Form | 2025 (legacy) | 2026+ | Notes |
|---|---|---|---|
| **1099-NEC** | $600 | **$2,000** | Payments on/after 2026-01-01; inflation-indexed from 2027 |
| **1099-MISC** | $600 | **$2,000** | Same change |
| **1099-K** | (in flux) | **$20,000 + 200 txns** | Restored by OBBBA |
| Backup withholding | $600 | **$2,000** | Aligns with new level |

Per-contractor, per-year. Texas has no state income tax, but **state rules vary** — store per-state thresholds if you ever pay PMs outside TX, and watch states that statically codify $600 (e.g., MS, WI) or conform to federal (e.g., CA adopted $2,000 for 2026).

> Even below threshold, the payment is still a deductible business expense for you and taxable income to the PM. Collect W-9s from **all** PMs regardless of expected volume.

## W-9 collection

Collect at onboarding, before first payout — don't chase data in January.

| Field | Maps to |
|---|---|
| Legal name | W-9 line 1 |
| Business name (if any) | line 2 |
| Federal tax classification | box 3 (individual/sole-prop/LLC/S-corp/C-corp/partnership) |
| TIN (SSN or EIN) | Part I — collected by provider, never stored raw here |
| Address | Part I |
| Signature + date | Part II |

Store only `tin_last4`, `tin_match_status`, provider record id, and the W-9 PDF key in R2.

## TIN matching

Validate name+TIN against IRS records via the filing provider's real-time TIN check **before** filing season. Statuses: `match`, `mismatch`, `foreign`. A `mismatch` triggers backup-withholding consideration and a re-collection request.

## Provider options

| Provider | Strengths |
|---|---|
| **Tax1099** | API for W-9 collection, real-time TIN checks, tax-exempt checks, e-file 1099 + 1042-S |
| **Track1099 (Avalara)** | API e-file workflows, embedded W-9 / W-8BEN / W-8BEN-E |
| **IRS IRIS** | Direct e-file; Taxpayer Portal supports up to 100 returns at a time (manual or CSV) |

**Build order for filing:**
1. **1099-ready export first** (CSV that any provider or a CPA can ingest).
2. **Tax1099 or Track1099 API** integration second.
3. **Direct IRS IRIS** only later, if volume justifies owning the filing.

> e-file is mandatory if you have **10+ information returns total** in a year (all 1099s/W-2s/1098s combined). Plan for e-file from the start.

## Eligibility computation

```python
def compute_1099_eligibility(pm_user_id, tax_year):
    paid = Ledger.pm_ytd_paid(pm_user_id, tax_year)   # completed payouts only
    threshold = tax_thresholds.get(tax_year, '1099-NEC').threshold_cents
    eligible = paid >= threshold
    pm_payout_profiles.update(is_1099_eligible=eligible)
    return {'paid_cents': paid, 'threshold_cents': threshold, 'eligible': eligible}
```

Run continuously (so the operator sees who's approaching threshold) and finalize after year-end close.

## Double-reporting guard

If a payment rail itself issues a 1099 (e.g., a 1099-K), **do not also issue a 1099-NEC for those same payments** — that double-reports the PM's income. Tag each payout with its rail; only payments through rails that do **not** issue their own form roll into your 1099-NEC totals. With the Dwolla ACH path, Acme Finance is the payer of record and issues the 1099-NEC; document this clearly.

## Schedule C / CPA packet export

Year-end packet, generated from ledger attributions:

- **Income:** total revenue (PayPal-reconciled), merch revenue.
- **Expenses by Schedule C line:** payouts (contract labor), materials, vehicle/mileage, processor fees, COGS, etc. — driven by `ledger_lines.tax_category`.
- **Contractor summary:** per-PM YTD paid, W-9 status, TIN-match status, 1099 eligibility.
- **Mileage summary:** total miles × rate by year (see `09-MILEAGE-EXPENSES.md`).
- **Attachments index:** receipt R2 keys per expense.

Export formats: CSV (provider ingest), PDF (CPA review), JSON (programmatic).
